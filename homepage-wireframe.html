Architecture notes (not on-page)

The Earth-side / Wittyverse-side toggle is not surfaced on the homepage in this hybrid. The IA still needs to accommodate it — most likely it lives on the About page or appears as a subtle "side door" element accessible from the masthead. The website brief requires the architecture to support the toggle even when it's not foregrounded.

Voice register split applies sitewide. Homepage default surfaces (hero, browse tiles, social proof, footer) read in third-person — the shop describes itself. About + service surfaces (404, cart, refund emails, order confirmations) read in first-person Winston.

Mobile collapse behavior is noted per section in each block. The wireframe at 900px and below stacks the notes column under the layout column and collapses multi-column slot grids.

What this wireframe is for: evaluating the homepage's section list + each section's function + the unique-Wittyverse features it carries. Visual decisions (palette, type, motion, illustration vs photo, layout fidelity) are deliberately deferred.
