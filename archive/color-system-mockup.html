Tier 2

Semantic tokens

Named by what they do. These are the tokens almost every var(--…) reference in styles.css should use. They map to Tier-1 globals and re-map per surface mode.

Surfaces

→ var(--cream)
Chrome surface
--surface-chrome
Hero, marquee, brand-chrome moments. Default mood = "lights down, neon up."
→ var(--purple-400)
Elevated
--surface-elevated
Cards on cream. Higher z-elements that need separation.
→ var(--white)
Inverse
--surface-inverse
Dark surface inside a light context. Footers, callouts, accent rows.
→ var(--ink)

Attention surface. Banners, "new" callouts, marquee CTAs.

Tier 3

Component tokens

Sparse. Only created when a specific combination is repeated across components. The --card-shadow and --cta-shadow-stack tokens encode the brand's signature "hard offset shadow" language as reusable values.

§4

The two modes, side by side

Wittyverse has two explicit surface modes — never OS-auto-flipped. They live as data attributes on a section wrapper, and semantic tokens re-map automatically inside.

Chrome mode

Hero, marquee, social card. The default "lights down, neon up" Wittyverse mood. [data-surface="chrome"] on the wrapper.

DELIVERED FROM CYCLE 247
Card on chrome

Long-form mode

Article, doc, product page. Cream-anchored for reading rhythm. Default for any surface where text exceeds 200 words. [data-surface="long-form"] on the wrapper.

SECTION HEADER
Card on long-form
Why explicit modes, not auto-flip? The brand has a default mood — purple chrome. Auto-flipping based on OS dark-mode setting would lose that. Modes are chosen per surface by the page author (hero = chrome, article = long-form), so a single page can carry both.

§6

Interactive states

Every action token has five states: default · hover · active · focus · disabled. Hover and active are color-shifted; focus adds a ring; disabled neutralizes everything. The signature hard-shadow language behaves consistently across states — hover compresses the stack, active pushes the button into the shadow.

Primary
action-primary · lime-on-ink + stacked shadow
default
bg ink · fg lime-400 · shadow 4px ink + 8px purple-700
hover
bg ink-lifted · fg lime-100 · shadow compresses to 2px+4px · translates −2,−2
active
fg lime-700 · shadow flattens to 0 · translates +4,+4 (pressed-into-shadow)
focus
+ 3px lime-400 ring · offset 4px · keyboard nav only
disabled
bg gray-300 · fg gray-500 · shadow gray-500 · cursor not-allowed
Secondary
action-secondary · ink-on-cream
default
bg cream · fg ink · shadow 4px ink
hover
bg cream-warmer · fg purple-700
active
bg gray-100 · shadow flattens
focus
+ 3px purple-400 ring
disabled
bg gray-300 · fg gray-500
Accent
action-accent · ink-on-orange + double shadow
default
bg orange-500 · fg ink · shadow 4px ink + 8px purple-700
hover
bg orange-lifted (~#FF8050) · translates −2,−2
active
bg orange-700 · shadow flattens
focus
+ 3px purple-700 ring
disabled
bg gray-300 · fg gray-500
The behavior pattern (consistent across all three actions): hover lifts the button up-and-left into its shadow (translate −2,−2 + shadow compresses) — the brand's "the button is jumping toward you" motion. Active presses down-and-right into the shadow (translate +4,+4 + shadow flattens to 0) — physical-feeling response. Focus adds a 3px ring offset 4px in a high-contrast hue (lime / purple). Disabled neutralizes everything to gray-300/gray-500 with no shadow color.
§7

Contrast matrix · pairings

Every meaningful text-color × surface-color combination. Ratios are computed live from the hex values in :root — what you see is what the system actually delivers. AAA ≥ 7:1 (preferred for body) · AA ≥ 4.5:1 (acceptable for body) · AA-large ≥ 3:1 (large text only, ≥18px) · FAIL < 3:1 (decorative only).

Approved pairings — by surface

The visual answer to "what text colors can I use on this surface?" Filtered to AA+ only.

Read this matrix before opening a PR. The rule: body copy must be AA or better; UI labels must be AA-large or better; decorative type only may go below. When in doubt, drop to a darker variant (e.g., orange-500 → orange-900 for text on cream) and re-check.
