Wittyverse.com — PDP Wireframe

Second wireframe in the series. Plain structural mockup — focus is on sections, functions, and unique-Wittyverse features on the product detail page. Three stance decisions are locked: (1) restraint — standard DTC PDP skeleton with one or two world-bearing beats woven in, parallel to the homepage's hybrid; (2) the gallery thumbnail strip carries one leak image slot (Wittyverse-side photo of the product in its native context); (3) a full-width origin card sits below the long description as the explicit world-bearing beat. Visual treatment intentionally absent.

Canonical world-bearing beat on the PDP. Lore-justified per §7.5: origin = portal-imported documentation; presentation = standard product context card. Narratively paired with the leak image in §4 — the copy can explicitly reference it ("pictured above on the depot shelf, this unit cleared through O.O.P.S. records on Cycle 247.3"). Mirrors the homepage's §6 "At the Counter" world-bearing slot in structure (eyebrow + copy + image) but pitched to a product, not the operator.

Architecture notes (not on-page)

Two world-bearing beats, restraint maintained. The PDP carries (1) the leak image in the §4 gallery thumbnail strip and (2) the §6 origin card. The rest of the page reads as a standard DTC PDP — gallery + info column + long description + reviews + related. This mirrors the homepage's "one world-bearing section in an otherwise lean commercial shape," scaled down to product level.

Voice register. The PDP is a default storefront surface → third-person shopkeeper throughout (eyebrows, product title context, long description, origin card, trust line, related-products heading). First-person Winston is reserved for service surfaces (About, 404, cart, refund emails, order confirmations). The origin card in particular reads institutionally — "museum wall label," not "Winston's note."

Earth-side / Wittyverse-side toggle. Not surfaced on the PDP as a foreground element. The IA still needs to accommodate it (per website brief), but on the PDP the WV-side content is reached through the leak image, the optional WV-side sub-title, the WV spec rows, and the origin card — not via a top-level toggle.

Lore-justification (§7.5 discipline). Every world-bearing element on the PDP traces to either a standard UI affordance or a Wittyverse-canon cause: the leak image = portal leakage (homepage §9); the WV spec rows = O.O.P.S. / depot documentation traveling with the shipment; the origin card = portal-imported product documentation, presented in standard museum/receiving-record format. Decorative weirdness is banned.

Mobile collapse behavior matches the homepage: 900px breakpoint stacks the notes column under the layout column and collapses multi-column slot grids (pdp-layout, origin-card-layout, grid-4, grid-3).

What this wireframe is for: locking the PDP's section list, each section's function, and the unique-Wittyverse features it carries. Visual decisions (palette, type, motion, photography style, gallery interaction patterns, illustration vs photo for the origin card image, exact eyebrow copy) are deliberately deferred to the prototype phase.
