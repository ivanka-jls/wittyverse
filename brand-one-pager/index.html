TL;DR

Two registers, one toolkit

Wittyverse runs on a deliberate split. Know which register you're designing for before you pick a color or a typeface — they share tokens but live by different rules.

R2 — Storefront chrome
Winston the importer
Loud, showman, candy-bar / patent-medicine / theme-park signage. Saturated polychrome by design. This is what most of the brand surfaces — hero, marquee, social, ads, product pages — live in.
R1 — In-universe canon
Wittyverse content
Deadpan, bureaucratic, customs-paperwork. Restrained editorial register — governed by the brand bible, not this doc. The joke lives in the proximity of R2 and R1, not in either one shouting.
One rule above all: Code reads Tier 2 / Tier 3 semantic tokens (--surface-default, --action-primary-bg, --card-shadow) — never raw --purple-400 or hex values. Designers can browse the raw values for reference, but everything wired into product UI flows through the semantic layer.

Six usage rules

  1. 1
    One saturated lead per screen.Lime or orange leads. The other plays support. Two competing peak hues = competing carnivals.
  2. 2
    Purple has two jobs.purple-400 for hero / marquee / CTA shadow. purple-700 for sustained chrome (long sidebars, dashboards). Don't dwell on -400 surfaces.
  3. 3
    Cream is the data ground.Dense tables, grids, repeating cards → cream surface. Saturated palette is for chrome / accents / single moments — never data layers.
  4. 4
    -100 tints carry brand DNA into dense layouts.Card backings, callouts, badges-at-rest use -100. Reserve saturated -400 / -500 for attention.
  5. 5
    Max one stacked-shadow per section.The signature lime+orange stack carries the focal point. Two within 600px muddies the eye. Pick the CTA or the hero, not both.
  6. 6
    State green / brand lime — keep apart in dense data.Distinct hue families (135° vs 85°), but in tight badge columns prefer spatial separation — split brand tags and status into two columns.

Two grounds — when to use which

Cream-ground · long-form

Cream is the GROUND. White is what LIFTS off it (cards, inputs, modals). Ink for body text, brand colors as accents only.

  • ✓ Blog · docs · product pages · checkout
  • ✓ Tables · dashboards on internal admin views
  • ✗ Hero / marquee / dark-mode social

Chrome-ground · loud

Purple as ground, cream for running text, lime for short labels and links. Apply via [data-surface="chrome"].

  • ✓ Hero / marquee · social / ads · dark mode
  • ✓ Showman moments — single dramatic spread
  • ✗ Long-form reading · dense data · forms

Typography

Five faces, five roles

Each face has one job — don't cross them. Lilita One is the wordmark and the wordmark only. Big Shoulders Display carries all display copy on the page. Bowlby One is the secondary "pulp display" face — reserved for hero banners, marquee strips, and ad headlines, max one use per page. Public Sans (US Web Design System) handles body — civic-tech humanist register that reinforces Winston's "authorized importer" framing. JetBrains Mono is reserved for the regulatory / CYCLE 247 microline that lets R1 peek through R2.

Wordmark · Lilita One
WITTYVERSE
Locked to the wordmark and the W monogram. ALL CAPS, skewX(−6deg). Canonical stacked colour shadow: orange fill · ink (4px offset) · lime (8px offset) — same stack across wordmark and monogram. Never use Lilita One for display copy, body, or accents.
Display · Big Shoulders Display 900
DELIVERED FROM CYCLE 247
All page display copy — H1 through H4, card titles, CTA labels, eyebrow displays. Skew range −5° to −7°. Pulp-mag swagger; never used at body sizes.
Pulp display · Bowlby One · max 1 use per page
ASTOUNDING NEW IMPORTS
Reserved for the loudest moment on a page — hero banners, marquee strips, social ad headlines. Same canon stack as the wordmark: orange fill · ink (3px) · lime (6px back-shadow). Skew −7°. Pairs as a poster sibling to Big Shoulders Display. Never use for body, eyebrows, CTAs, navigation — and never more than once per page or it dilutes the signal.
Body · Public Sans 400 / 500 / 600 / 700 / 800
Public Sans sets the body, the labels, and the workhorse copy. The quick brown fox jumps over the lazy dog 0123456789.

Casing & skew rules

ALL CAPS where wordmark, display headings (H1–H4), eyebrows, CTAs, mono tags. Body and links stay sentence-case.
Skew range −5° to −7° for wordmark, monogram, and display headings. Below −4° loses punch; above −7° gets unstable.
Never mix display + display in a single composition. Pair Big Shoulders display headline with Public Sans subline.
Composition

Layout do & don't

The visual language is asymmetric, hard-edged, and confidently polychrome. Soft drop-shadows on their own, hairline borders, and justified body text all read as "default web template" — which is the trap.

Do

  • Compose asymmetrically — off-centre hero CTAs, tilted display type, headlines that break the grid.
  • Pair hard ink shadow (0 4–8px 0 ink) with the brand stack — that's the signature depth.
  • Layer flat colour blocks over the purple-gradient backdrop in chrome surfaces.
  • Use ✦ star and pipe (|) as decorative separators in brand chrome.
  • Reserve uppercase for wordmark, display headlines, and eyebrows — body stays sentence-case.

Don't

  • Don't use soft drop-shadows alone — always pair with the hard offset.
  • Don't justify body copy or centre long-form text — always left-align.
  • Don't use hairline borders — minimum 2px, brand standard 3–4px.
  • Don't auto-invert via OS preference — the brand has a default mood per surface type.
  • Don't reach for raw --purple-400 in product code — go through the semantic token layer.
