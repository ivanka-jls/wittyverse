Wittyverse.com — PDP Wireframe

Second wireframe in the series. Hybrid layout: each section renders as it would on the real page (full width, sub-elements visible and proportional), with a notes block visually offset below carrying the function description, ◆ WV feature callouts, and decisions to make. Three stance decisions are locked: (1) restraint — standard DTC PDP skeleton with one or two world-bearing beats woven in, parallel to the homepage's hybrid; (2) the gallery thumbnail strip carries one leak image slot (Wittyverse-side photo of the product in its native context); (3) a full-width origin card sits below the long description as the explicit world-bearing beat. Visual treatment intentionally absent.

Architecture notes (not on-page)

The PDP carries two world-bearing beats (leak image in §4 gallery + origin card in §6) — restraint maintained, parallel to the homepage's single explicit world-bearing section. The rest of the page reads as a standard DTC PDP with foreign-fingerprint accents in copy.

Voice register: the PDP is a default storefront surface → third-person warm shopkeeper throughout. First-person Winston is reserved for service surfaces (About, 404, cart, refund emails).

The Earth-side / Wittyverse-side toggle is architecture-only on this page; not surfaced as a foreground element. The IA still accommodates it (the leak image + origin card prove the WV side exists without requiring a visible switch).

Mobile collapse behavior is noted per section in each block. The wireframe at 900px and below collapses the gallery + info-column grid, the origin card grid, the notes block to a single column, and the multi-column slot grids.

What this wireframe is for: locking the PDP section list + each section's function + the unique-Wittyverse features it carries. Visual decisions (palette, type, motion, illustration vs photo, layout fidelity) are deliberately deferred to the prototype phase.
